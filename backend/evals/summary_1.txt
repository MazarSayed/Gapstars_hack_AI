Topics

LinkedIn URL Normalization

  - Problem: Scraped LinkedIn company URLs contain query parameters (e.g., ?trk=...) that are unnecessary for API calls and can cause lookup failures.
  - Solution: A new function removes all characters after and including the first ? in a URL.
  - Rationale: The core URL (e.g., linkedin.com/company/company-name) is a unique identifier, making the query string redundant.

Enrichment Strategy & API Validation

  - Goal: Find the correct decision-maker for a given job role and company.
  - Challenge: Company names are not unique (e.g., "PTY LTD" variations), making them unreliable for API lookups.
  - Solution: Use the unique LinkedIn company ID as the primary API input.
  - API Validation Plan:
      - Test Apollo's enrichment API on 20–25 companies.
      - Compare results against alternatives like Lusha to select the best provider.
      - Document findings to support the final decision.
  - Cost-Effective Enrichment Logic:
      - Rationale: Avoid burning API credits on low-quality contacts.
      - Process:
        1.  Filter initial API results using Apollo's email_status (e.g., verified) and has_phone flags.
        2.  Rank remaining contacts by relevance (e.g., role match, contact availability).
        3.  Fetch full details only for the top 1–3 highest-confidence contacts.

Workflow & UI Design

  - Architecture: The system will be built as a set of individual functions ("tools") that can later be integrated into an agentic workflow.
  - Human-in-the-Loop Process:
    1.  Scrape: Daily job scrapes populate a lead database.
    2.  Enrich: The system finds decision-makers and drafts personalized emails.
    3.  Review: Leads and drafts are pushed to a UI (e.g., Streamlit app, Google Sheet) for manual approval.
    4.  Send: Approved emails are sent.
  - Future Enhancements (V2):
      - Email Tracking: Track opens and clicks via deep links.
      - Bounce Handling: Catch SMTP delivery failures.
      - Follow-up Cadence: Implement multi-touch email sequences.
